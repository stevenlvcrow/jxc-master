# 前端开发统一标准（最终简化版）
## 一、核心原则
1. **真实优先**：默认接真实接口，临时 mock 必须标注、独立文件、联调前移除
2. **作用域必传**：所有接口必须携带 `sessionStore.currentOrgId`，切换机构自动刷新
3. **权限闭环**：前端只控制按钮展示，最终权限以服务端接口返回为准
4. **结构统一**：以物品管理页为骨架，禁止复制缺陷实现

## 二、页面结构标准
### 1. 列表管理页
固定结构：左侧树（可选）→ 查询区 → 工具栏 → 表格区 → 分页区

### 2. 树表管理页
左树仅做筛选，点击必须触发右侧列表重新查询

### 3. 页面内多页签
使用 `PageTabsLayout.vue`，各页签状态独立、互不污染

### 4. 复杂新增/编辑页
新增/编辑共用一套表单与校验逻辑，编辑必须调用详情接口回填

## 三、组件职责
1. **页面容器**：唯一状态源，管理查询、分页、接口、权限、作用域
2. **子组件**：仅做展示与事件抛出，禁止调用 API、禁止修改父组件数据
3. **禁止行为**：子组件不维护业务状态、不读写全局状态、不拼接参数

## 四、接口与数据规范
1. 接口统一存放：`src/api/modules/`
2. 分页参数：`pageNo/pageSize`，查询重置到第一页
3. 编码规则：物品/分类/规则编码由后端生成，前端仅展示
4. 编辑流程：查详情 → 回填 → 保存 → 刷新列表
5. 异常处理：所有接口 `try/catch`，401 自动登出，403 提示权限

## 五、机构与权限
1. 未选择机构禁止进入业务页面
2. 菜单由接口返回，前端不伪造、不推导管理员
3. 按钮权限：页面级控制 + 接口校验双重保障
4. 切换机构：清空旧数据 → 重新加载菜单 → 重新查询列表

## 六、代码与目录规范
```text
src/views/模块/
  management.vue      列表页
  create.vue          新增/编辑页
  types.ts            类型定义
  management-data.ts  静态常量
  components/
    XxxQuerySection.vue    查询区
    XxxToolbarSection.vue  工具栏
    XxxTableSection.vue    表格
    XxxPaginationSection.vue 分页
```

## 七、交互强制规范
1. 查询 → 回到第一页
2. 重置 → 清空条件并重查
3. 删除/批量删除 → 二次确认
4. 提交 → 防重复提交、成功提示、刷新数据
5. 空数据/无权限/接口失败 → 分开提示

## 八、提测验收标准
1. 无 mock、无假数据、无硬编码机构
2. 全接口携带 `orgId`，切换机构数据正确
3. 新增/编辑/删除/状态切换全链路通
4. 多角色、多机构权限验证通过
5. 无 `console.log/debugger`，无 `any` 类型
