# 前端开发统一标准（以物品管理页为基线）

## 1. 文档目标
- 本文档用于统一 `jxc-w` 的后台页面开发方式，重点解决当前项目中已经暴露出的四类问题：
  - 页面长得像已完成，但后端仍是假数据或混合数据源。
  - 菜单看起来有权限控制，但实际能力校验不闭环。
  - 页面能切换机构，但查询/新增/编辑没有把机构作用域真正带到接口。
  - 页面只做静态构建，不做真实链路验收，导致“能 build 不代表能上线”。
- 本文档不是纯 UI 规范，而是“页面结构 + 状态管理 + 接口接入 + 权限与作用域 + 联调验收”的统一标准。
- 适用范围：`jxc-w/src/views/**` 下所有列表页、树表页、复杂新增页、权限管理页。
- 物品管理页仅作为页面骨架母版，不再作为“数据实现方式”母版。

## 2. 基本原则

### 2.1 真实优先
- 新页面默认接真实接口，不允许先以内存常量、伪分页、硬编码总数作为长期方案。
- 如因后端未完成必须临时 mock，必须同时满足：
  - 页面顶部或开发说明中明确标注“当前为 mock 数据”。
  - mock 数据放入独立文件，不得混入页面主逻辑。
  - 接口函数签名、字段结构、分页模型必须与未来真实接口一致。
  - 联调前必须移除 mock 分支，不允许带着“双轨逻辑”长期存在。

### 2.2 作用域显式
- 只要页面属于集团/门店维度业务，查询、新增、编辑、删除、导出等所有接口都必须显式带当前 `orgId`。
- `orgId` 统一来源于 `sessionStore.currentOrgId`，禁止组件内部自己拼装另一套机构状态。
- 页面切换机构后，必须重新加载当前页数据，不允许继续展示上一个机构的数据快照。

### 2.3 权限闭环
- 前端菜单只负责导航，不代表真实权限。
- 页面能否展示某个按钮，必须满足两个条件：
  - 菜单或页面能力允许展示。
  - 实际接口返回未被服务端拒绝。
- 禁止把“前端隐藏按钮”当作权限实现。

### 2.4 母版复用但不复制缺陷
- `src/views/items/management.vue` 仍是列表页视觉和结构母版。
- 但历史页面中的任何临时写法，包括硬编码数据、弱作用域透传、把业务逻辑塞进子组件，不得继续复制。

## 3. 页面分型标准

### 3.1 列表管理页
- 典型页面：物品管理、用户管理、角色管理、门店管理、供应商管理。
- 固定骨架顺序：
  1. 左侧树区或分类区，可选
  2. 查询区
  3. 工具栏
  4. 表格区
  5. 分页区

### 3.2 树表管理页
- 典型页面：物品分类、供应商分类、菜单权限。
- 左树仅负责“当前筛选上下文”，右侧列表才是实际数据结果。
- 左树节点点击后，必须触发右侧查询参数变化，而不是只改本地高亮。

### 3.3 页面内多页签页
- 典型页面：菜品关联仓库、费用关联仓库、同一业务下的多视角配置页。
- 页面内多页签是“单页面内部的二级工作区切换”，不是主框架顶部工作台页签。
- 固定骨架顺序：
  1. 页面内多页签头
  2. 当前页签内容区
  3. 页签内容区内部再按各自页面类型套用查询区、工具栏、树区、表格区、分页区标准
- 多页签之间允许共享一个页面容器，但每个页签必须维护独立的查询参数、分页、勾选项和表格数据，禁止两个页签共用同一份查询状态互相污染。
- 若某个页签带树区，树区只作用于当前页签，不得影响其他页签的筛选上下文。

### 3.4 复杂新增/编辑页
- 典型页面：新增物品、新增供应商。
- 固定骨架顺序：
  1. 固定操作头
  2. 面包屑
  3. 主表单
  4. 分区块
  5. 底部操作区或固定头主操作
- 新增页和编辑页必须共用一套表单模型与校验逻辑，禁止复制两套实现。

### 3.5 权限配置页
- 典型页面：菜单权限管理、角色管理。
- 这类页面必须把“配置结果是否真正生效”纳入验收，而不是只完成勾选树和保存接口。

## 4. 页面骨架与 UI 基线

### 4.1 布局基线
- 双栏布局默认：`grid-template-columns: 120px minmax(0, 1fr)`。
- 内容区内边距：`8px 12px 12px`。
- 主面板内边距：`8px`。
- 间距：查询区、工具栏、表格、分页之间保持 `6px` 节奏。
- 禁止页面各自再发明新的主容器命名与尺寸体系。
- 禁止使用页面“额头标题块”组件模式：`<div class="section-head"><strong>...</strong></div>`。
- 页面标题统一由路由与侧边菜单承载，页面正文从查询区/工具栏直接开始；如需补充说明，使用普通文本说明区，不使用 `section-head` 样式容器。
- 页面内多页签统一使用 `src/components/PageTabsLayout.vue`，禁止各页面自行拼一套页签头、激活态和内容容器。
- `PageTabsLayout` 只负责页签切换骨架、右侧操作位和正文插槽，不负责业务查询、接口请求和页签内状态管理。
- 页面内多页签的激活态颜色必须跟随项目主色 `var(--brand)`，禁止单页改成黄色或其他脱离全局体系的强调色。

### 4.2 查询区基线
- 查询区只承载筛选项和交互事件，不承载请求逻辑。
- 查询区优先复用 `src/components/CommonQuerySection.vue` 或业务模块 QuerySection，禁止页面长期手写分散式 `el-form` 作为默认方案。
- 输入框、下拉、按钮高度统一使用 `22px` 体系。
- 点击“查询”必须重置到第一页。
- 点击“重置”必须恢复默认筛选、默认树节点、默认分页，并重新拉取数据。
- 查询与重置按钮属于查询区，不属于工具栏；工具栏只放批量动作、导入导出、跳转、新增等操作入口。

### 4.3 工具栏基线
- 工具栏只负责动作入口，不承担业务状态计算。
- 工具栏优先复用 `src/components/CommonToolbarSection.vue` 或业务模块 ToolbarSection，禁止每个页面重复手写一组结构相同的按钮容器。
- 首按钮默认主按钮。
- 禁止把“是否允许点击”的复杂权限判断散落在按钮模板里，统一收敛成计算属性或页面级能力对象。

### 4.4 表格基线
- 表格默认字体 `10px`，行高 `20px` 体系。
- 列表页表格必须支持以下三种状态：
  - 首次加载
  - 空数据
  - 接口失败后恢复
- 操作列文案和顺序在同类页面中保持一致，例如“编辑 / 启停 / 删除”。

### 4.5 分页基线
- 分页组件只负责页码和页大小变更。
- 删除后若当前页已无数据，页面容器必须自行回退到上一页并重查。

### 4.6 弹窗表单基线
- 小弹窗：`420px ~ 480px`
- 常规弹窗：`520px ~ 560px`
- 复杂弹窗：`760px`
- 同一弹窗内标签宽度必须一致，通常为 `90px` 或 `100px`。
- 必填校验使用统一规则源，禁止在提交前再手写一遍 if/else 校验。

## 5. 组件职责标准

### 5.1 页面容器职责
- `management.vue`、`create.vue` 等页面容器统一承担：
  - 查询参数管理
  - 当前机构作用域注入
  - 接口请求
  - 加载态
  - 错误恢复
  - 路由跳转
  - 批量操作
  - 成功提示
- 页面容器必须是“单一真实状态源”，禁止子组件保存第二份业务真状态。

### 5.2 子组件职责
- `QuerySection`：只收集筛选条件和抛出事件。
- `ToolbarSection`：只承载按钮展示和动作抛出。
- `TableSection`：只承载表格展示、行事件、勾选事件。
- `PaginationSection`：只承载分页展示和变更事件。
- 左树组件：只承载节点渲染、选中态和节点事件。
- `PageTabsLayout`：只承载页面内多页签头、激活态切换、右侧插槽和正文容器；页签内业务状态仍由页面容器统一管理。

### 5.3 禁止事项
- 子组件禁止直接调用业务 API。
- 子组件禁止直接读写 `sessionStore.currentOrgId` 以外的全局业务状态。
- 子组件禁止偷偷修正父级业务参数。
- 子组件禁止维护与父组件重复的分页、查询、选中项真状态。

## 6. 数据与接口接入标准

### 6.1 API 模块划分
- 接口定义统一放在 `src/api/modules/**`。
- 页面中禁止直接拼接 URL 调 `apiClient`。
- 每个业务模块至少包含：
  - 查询列表
  - 查询详情
  - 新增
  - 更新
  - 状态变更
  - 删除或批量删除

### 6.2 请求参数标准
- 作用域型业务统一使用 `orgId` 参数。
- 列表分页统一使用 `pageNo/pageSize`。
- 查询区筛选项为空时，传参与不传参必须有统一策略：
  - 能不传则不传。
  - 不传和传空串禁止混用。

### 6.3 返回数据标准
- 列表页必须基于真实返回结构渲染，不允许在页面里伪造总数、序号、状态映射后冒充后端数据。
- 若后端返回的是枚举值，前端只做展示映射，不得私自发明新业务值回写。

### 6.4 编辑链路标准
- 进入编辑页时，必须通过详情接口回填，不允许依赖列表行残缺字段。
- 编辑保存后返回列表时，必须明确刷新来源页数据。

### 6.5 编码字段标准
- `规则编码`、`物品编码`、`分类编码` 统一由服务端生成，前端禁止提供手工录入入口。
- 新增页、批量新增弹窗中，编码字段只允许展示，不允许作为必填校验项。
- 提交新增请求时，前端不得依赖本地规则拼编码；若请求对象保留编码字段，必须传空或不传。
- 只有产品明确提出“允许手填编码”时，才可恢复输入框，并且必须在需求说明中注明。

## 7. 机构与权限标准

### 7.1 机构选择
- 需要机构作用域的账号必须先选机构，再进入业务页。
- 页面不得在未选择机构时偷偷以平台级或默认门店继续请求。
- 对 `orgId` 非法或缺失的情况，页面必须明确阻断并提示，而不是静默兜底。

### 7.2 菜单与路由
- 菜单接口返回什么，前端就渲染什么；前端不补菜单、不猜菜单、不伪造管理员菜单。
- 路由守卫只负责：
  - 未登录跳登录
  - 未选机构跳机构选择
  - 当前页面不在允许菜单中时回首页
- 路由守卫禁止根据账号字符串自行推导“管理员模式”。

### 7.3 按钮权限
- 若后端暂未提供按钮级能力接口，前端允许先按页面级菜单显示按钮。
- 但保存、删除、状态变更等写操作必须以服务端返回为准，收到权限错误时必须展示明确提示。

### 7.4 权限配置页特殊要求
- 配置类页面提测前必须验证：
  - 保存角色/菜单关系成功
  - 重新登录或刷新菜单后权限变化真实生效
  - 非授权用户不能通过直接访问 URL 进入目标页面

## 8. 状态管理标准

### 8.1 Session Store
- `sessionStore` 只负责：
  - 登录态
  - 当前用户显示信息
  - 当前机构树
  - 当前机构 ID
- 禁止把“是否平台管理员”写成基于账号名的前端推导规则。
- 平台管理员模式若存在，必须来源于后端明确返回的用户能力，而不是本地猜测。

### 8.2 401 处理
- 刷新 token 失败时，必须同时清空 token、清空登录态、清空机构、清空菜单并跳回登录页。
- 禁止出现“本地还显示已登录，但所有接口都 401”的状态撕裂。

### 8.3 菜单 Store
- 菜单必须跟随当前机构维度加载。
- 切换机构时，必须清理旧菜单并加载新菜单。
- 菜单接口失败时，不得继续展示上一个机构菜单冒充当前机构权限。

## 9. 代码组织标准

### 9.1 目录模板
```text
src/views/<module>/
  management.vue
  create.vue                  # 有新增/编辑页时
  management-data.ts          # 仅允许放纯静态展示常量，不放伪业务数据
  types.ts
  components/
    <Module>QuerySection.vue
    <Module>ToolbarSection.vue
    <Module>TableSection.vue
    <Module>PaginationSection.vue
    <Module>CategoryTree.vue
```

### 9.2 命名规则
- 列表页：`management.vue`
- 新增页：`create.vue`
- 编辑页优先与新增页共用 `create.vue`，通过路由参数区分。
- 组件命名：`<业务名><功能段>Section.vue`
- 类型定义统一进入 `types.ts`，禁止在多个组件内各写一份重复 type。

## 10. 交互标准
- 查询必须回第一页。
- 重置必须清空筛选并重查。
- 批量操作无勾选必须提示。
- 单删和批删必须二次确认。
- 保存成功后必须有明确反馈，并更新当前页状态。
- 提交按钮在请求中必须防重复提交。
- 空列表、无权限、接口失败三种状态必须区分，不能都显示“暂无数据”。

## 11. 响应式标准
- `>= 1200px`：优先双栏。
- `< 1200px`：复杂表单可降为 2 列。
- `< 960px`：页面统一降为单栏，表格保留横向滚动。
- 响应式只允许调整布局，不允许因为窄屏就删除关键功能入口。

## 12. 联调与提测验收标准

### 12.1 必做联调项
- 登录
- 机构选择
- 菜单加载
- 列表查询
- 新增
- 编辑
- 状态切换
- 删除或批量删除
- 401 失效恢复

### 12.2 权限验收项
- 用至少两个不同角色账号验证：
  - 菜单不同
  - 可进入页面不同
  - 写操作权限不同
- 验证直接输入 URL 时，无权限用户不会因前端漏拦而继续使用页面。

### 12.3 作用域验收项
- 用至少两个不同机构验证：
  - 切换机构后列表数据变化正确
  - 新增数据写入正确机构
  - 编辑和删除不会误作用到其他机构数据

### 12.4 提测前禁止项
- 禁止保留长期 mock 开关不说明用途。
- 禁止页面依赖默认假机构树作为真实联调结果。
- 禁止前端通过账号名判断管理员身份。
- 禁止只提供 UI 截图，不验证真实接口链路。

## 13. 物品管理页作为母版的正确用法
- 可复用：
  - 页面分区结构
  - 查询区、工具栏、表格、分页四段式拆分
  - 紧凑表格视觉样式
  - 列表页交互节奏
- 不可复用：
  - 任何伪分页或内存数据思路
  - 未带 `orgId` 的接口调用方式
  - 把业务逻辑塞进子组件的写法
